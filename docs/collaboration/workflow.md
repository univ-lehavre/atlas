# Comment on travaille ensemble

_(à rédiger pour un public non-expert dans une PR ultérieure)_

Cette page décrira le flux de contribution standard :

1. Création d'une branche depuis `main`
2. Commits suivant la convention [Conventional Commits](https://www.conventionalcommits.org/)
3. Pre-commit + pre-push hooks (cf. [Hooks Git](../quality/hooks.md))
4. Ouverture d'une Pull Request
5. Revue de code
6. CI verte (typecheck, lint, tests, security, build, docs)
7. Merge (squash ou merge-commit selon scope)

Pour les contributeurs externes : ouvrir une [issue](https://github.com/univ-lehavre/atlas/issues) suffit pour signaler un bug ou une remarque sur la documentation — pas besoin de savoir coder.
