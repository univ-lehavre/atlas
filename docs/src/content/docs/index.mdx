## Ce qui distingue ce dépôt

Atlas n'est pas qu'un assemblage de projets : c'est une **chaîne de qualité
partagée**, outillée et vérifiable. Voici ce qu'un contributeur y trouve — chaque
point renvoie à la page qui l'atteste.

<CardGrid>
  <Card title="Typé strict, de bout en bout" icon="seti:typescript">
    Tout le code est en TypeScript en mode **strict** : les erreurs de type
    sont attrapées avant l'exécution, pas en production.

    [Style de code →](/atlas/quality/code-style/)

  </Card>
  <Card title="Qualité et sécurité outillées" icon="approve-check">
    Chaque pull request passe une **intégration continue** (lint, types, build,
    audits) et des contrôles de sécurité automatisés (analyse statique,
    détection de secrets, revue de dépendances).

    [Pipeline CI →](/atlas/quality/ci-pipeline/) ·
    [Sécurité →](/atlas/quality/security/)

  </Card>
  <Card title="Une doc qui ne ment pas" icon="open-book">
    La documentation est **vérifiable** : les chiffres et exemples sont
    rattachés au dépôt et contrôlés, pas affirmés à la main.

    [Politique de documentation →](/atlas/quality/documentation/)

  </Card>
  <Card title="Des tests à plusieurs niveaux" icon="puzzle">
    Une **pyramide de tests à cinq niveaux** couvre tout le code exécutable,
    du test unitaire au test de bout en bout, avec des seuils de couverture
    chiffrés.

    [Tests →](/atlas/quality/tests/)

  </Card>
  <Card title="Un socle fonctionnel commun" icon="setting">
    Les briques partagées reposent sur **Effect**, un socle de programmation
    fonctionnelle qui rend les erreurs et les effets explicites et typés.

    [Programmation fonctionnelle →](/atlas/quality/code-style/) ·
    [Décision (ADR 0005) →](/atlas/decisions/0005-effect-pour-la-pf/)

  </Card>
  <Card title="Une carte du code toujours à jour" icon="list-format">
    Pour chaque paquet — son rôle, ce dont il dépend, qui le consomme.
    **Générée depuis le code**, donc fidèle à la réalité.

    [Carte des paquets →](/atlas/architecture/packages/)

  </Card>
</CardGrid>
